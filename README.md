# test-js
# 🧪 JavaScript Unit & Integration Testing (TestJS)

ეს მოდული ეძღვნება აპლიკაციის ფუნქციონალის ავტომატურ ტესტირებას. პროექტში დანერგილია ერთეულოვანი (Unit) და ინტეგრაციული (Integration) ტესტები, რათა უზრუნველყოფილ იქნას ბიზნეს ლოგიკის, API ენდპოინტებისა და ფუნქციების 100%-ით გამართული მუშაობა კოდის რეფაქტორინგის დროს.

---

## 🎯 რისი ტესტირება ხდება?

* **დამხმარე ფუნქციები (Utility Functions):** მონაცემთა ვალიდაციის, მათემატიკური გამოთვლებისა და ტექსტის ფორმატირების ლოგიკა.
* **API რექვესტები (Endpoint Testing):** ბექენდის მარშრუტების (Routes) შემოწმება (სწორი პასუხები, სტატუს კოდები 200, 400, 404, 500).
* **მონაცემთა ბაზის მოდელები (Database Mocks):** ბაზაში ინფორმაციის სწორად ჩაწერისა და წაკითხვის იმიტაცია.

---

## 🛠️ გამოყენებული ტექნოლოგიები და ხელსაწყოები

* **Jest / Vitest:** მთავარი სატესტო ფრეიმვორქი (Test Runner & Assertion Library).
* **Supertest:** ბექენდის (Express.js) ენდპოინტების დასატესტად HTTP მოთხოვნების სიმულაციისთვის.
* **Mock Data:** სატესტო მონაცემების ბაზა, რომელიც არ უცვლის რეალურ მომხმარებლებს ინფორმაციას.

---

## 📁 სატესტო ფაილების სტრუქტურა

ტესტები განთავსებულია სპეციალურ `__tests__` საქაღალდეებში ან აქვთ `.test.js` / `.spec.js` გაფართოება:
* **`utils.test.js`** - დამხმარე მეთოდების ტესტები.
* **`auth.test.js`** - რეგისტრაციისა და ავტორიზაციის (Login/Register) ლოგიკის შემოწმება.
* **`trips.test.js`** - ექსკურსიებისა და ჯავშნების ფუნქციონალის ტესტირება.
